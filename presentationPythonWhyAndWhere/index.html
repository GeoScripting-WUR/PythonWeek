CPython, Cython, IronPython and Jython

  • Python is more like an ecossytem
  • You program in Python and an interpreter runs your code
  • "Bellow" it, we can have C, JAVA or .NET

CPython

  • Normally referred as Python
  • Python language binds with C and runs C code
  • Basically CPython is a sort of friendly C

IronPython

  • Python binds to the .NET framework from Microsoft
  • You program python but you run .NET

Jython

  • Using the JAVA libraries/classes
  • You access JAVA content using Python

Cython

  • Dialect of Python
  • You declare variable variables
  • This helps code to speed up
  • Python
    • def divide(a, b)
 return a/b
  • Cython
    • def divide(int a, int b)
 return a/b

Cython - Compilation

  • We can cython code compile into a module
  • Use module in Python
  • Cython compiler generates C code

Cython - Speed up

  • Normally 30 to 60%
  • Loop and vector operations can be 10 to 100x faster
  • Keep in mind that executable code is the fastes possible

PyPY

  • Different Python interpreter
  • Just in Time compilation (like in JAVA)
  • Part of translated machine code is compiled
  • Speed up of loops

GeoProcessing

  • Normally libraries are written in C/C++
  • Python is perfert to connect to them
  • GDAL/OGR all coded in C++
  • GDAL/OGR easier to program in Python

GDAL

  • Geospatial Data Abstraction Library (GDAL)
  • 200 driver to read rasters
  • Mainly written in C/C++
  • Even more friendly Python libraries (rasterio) are based on GDAL

OGR

  • OGR Simple Features Library
  • Twin brother of GDAL but for vector data
  • 65 drivers
  • Fiona is based on OGR
